# Rapport - Page de Personnalisation

### Objectif
Créer une page de personnalisation complète permettant à chaque structure médicale de personnaliser l'application générique avec ses propres logos, couleurs, textes et paramètres de documents. Cette fonctionnalité transforme une application générique en une solution sur mesure pour chaque établissement de santé.
 


## Fonctionnalités par Onglet

### 1. Onglet Général (35 paramètres)
**Informations administratives et organisationnelles**

- **Informations de la structure** : Nom, adresse complète (rue, ville, code postal, pays)
- **Coordonnées** : Téléphone, email, site web
- **Informations légales** : Numéro d'agrément, NINEA, registre de commerce
- **Horaires d'ouverture** : Configuration par jour (ouvert/fermé, heures début/fin)
- **Localisation et formats** : Langue (FR/EN/AR), fuseau horaire, format date/heure, devise
 

### 2. Onglet Apparence (28 paramètres)
**Personnalisation de l'identité visuelle**

- **Identité visuelle** : Logo principal, favicon, titre de la page
- **Couleurs principales** : 7 couleurs (principale, secondaire, accent, succès, avertissement, danger, info)
- **Couleurs interface** : 5 couleurs (fond, surface, texte principal/secondaire, bordures)
- 

### 3. Onglet Documents  
**Personnalisation des documents médicaux**

- **En-tête et identité** : Logo documents, cachet cabinet, lieu par défaut, options d'affichage
- **Couleurs documents** : Principale, secondaire, bordures (indépendantes de l'interface)
- **Certificats** : Titre, texte introduction (variable `[NOM_MEDECIN]`), mention légale, footer, options
- **Ordonnances** : Titre, footer, options d'affichage
- **Format et style** : Police (6 options), taille, marges, largeur max, couleur fond
- **Informations complémentaires** : Footer général (variable `[DATE]`), options d'affichage coordonnées
